# Contributing to Kent

## Ветка и релизы

- main: стабильные релизы (теги X.Y.Z). Merge только через Pull Request после успешного CI и ревью.
- develop: интеграционная ветка. Все feature/hotfix ветки ответвляются от неё и попадают через PR.
- eature/*: функциональные ветки. После завершения — PR в develop.
- hotfix/*: срочные исправления для main. После слияния в main обязательно вмерджить изменения обратно в develop.

## Процесс Pull Request

1. Перед PR убедитесь, что локально проходят ./gradlew :app:assembleDebug и go build ./....
2. CI запускается автоматически для main и develop. PR не сливается, пока пайплайны не зеленые.
3. Требования к PR:
   - Минимум один аппрув от ревьюера.
   - Обновлены тесты/документация (если релевантно).
   - В описании указаны изменения и способ тестирования.

## Код-стайл и проверки

- Android: Ktlint/Detekt (при необходимости), Compose guidelines.
- Go: gofmt, staticcheck (запланировано).
- CI: GitHub Actions / GitLab CI должны быть зелёными перед merge.

## Безопасность

- Не коммитить секреты; используйте переменные окружения/Secrets.
- Уязвимости или security-issues отправляйте на security@kent.app.
